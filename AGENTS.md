# Repository Notes

- Use `uv` as the package manager for this repository.
- Prefer `uv sync` for setting up dependencies locally.
- Prefer `uv run ...` when running project-scoped commands.
- Keep Docker builds aligned with the local `uv` workflow.
- Do not use decorative comment banners (e.g. `# --- Section ---` or `# ====`). Let the code structure speak for itself.
